### 2️⃣ 进货管理模块 (新功能)

#### 核心功能

##### 进货单创建
```
操作流程:
1. 输入产品信息
   ├─ 产品名称
   ├─ 进货数量
   └─ 单位进货价

2. 选择供应商
   ├─ 供应商名称
   └─ 联系信息

3. 自动计算
   ├─ 总进货金额 = 数量 × 单价
   └─ 成本预算评估

4. 保存进货单
   └─ 状态: 待审核

示例:
产品: 护肤精油套装
数量: 50件
单价: ¥180/件
供应商: 美妆供应商A
总金额: ¥9,000
```

##### 进货状态管理
```
状态流转:
待审核 → 已下单 → 已送达 → 已完成

🟡 待审核
   ├─ 刚创建的进货单
   ├─ 等待管理员确认
   └─ 可编辑和删除

🔵 已下单
   ├─ 确认已向供应商下单
   ├─ 等待供应商发货
   └─ 可取消订单

🟣 已送达
   ├─ 货物已收到
   ├─ 等待入库确认
   └─ 准备入库

🟢 已完成
   ├─ 货物已入库
   ├─ 库存已更新
   └─ 记录完成
```

##### 统计信息
```
统计卡片:
┌─────────────────────┐
│ 总进货单数     15   │
│ 待审核          2   │
│ 已下单          3   │
│ 已送达          5   │
│ 进货总额   ¥52,600  │
└─────────────────────┘

关键指标:
1. 进货效率
   └─ 从待审核到完成的平均时间

2. 供应商可靠性
   └─ 按时交货率 / 质量反馈

3. 进货成本控制
   └─ 平均单价 / 成本趋势

4. 库存周转率
   └─ 进货→销售的平均周期
```

### 3️⃣ 智能比价模块 (新功能)

#### 核心概念

##### 价格三维对标
```
我们的价格 vs 市场均价 vs 竞争对手价

例子:
产品: 面膜套装
我们的价格:      ¥99
市场均价:        ¥88
竞争对手价:      ¥85

分析:
价格差异: +14元 (比市场均价高14%)
结论: 价格偏高，建议降价
```

#### 智能推荐机制

##### 优先级判断
```
🔴 高优先级 (价格差 > 15%)
   ├─ 特征: 价格明显偏高
   ├─ 影响: 严重影响竞争力
   ├─ 建议: 立即降价
   └─ 目标客户流失风险高

🟡 中优先级 (价格差 5-15%)
   ├─ 特征: 价格有偏高
   ├─ 影响: 中等竞争力
   ├─ 建议: 小幅调整
   └─ 目标平衡成本与销量

🟢 低优先级 (价格差 < 5%)
   ├─ 特征: 价格合理
   ├─ 影响: 有竞争力
   ├─ 建议: 保持或小幅优化
   └─ 目标维持当前优势
```

##### 定价建议
```
场景1: 价格偏高
推荐: 降价
实施:
  └─ 新价格 = 市场均价 - 5%
  └─ 预期: 销量提升 20-30%

场景2: 价格合理
推荐: 保持
实施:
  └─ 保持当前价格
  └─ 预期: 稳定销量和利润

场景3: 价格偏低
推荐: 小幅提价
实施:
  └─ 新价格 = 竞争对手价 - 2%
  └─ 预期: 利润率提升 8-12%
```

#### 使用流程

```
1. 系统自动分析
   └─ 每次进入模块时自动对标

2. 查看分析结果
   ├─ 价格对比三维图
   ├─ 价格差异百分比
   └─ 优先级等级

3. 阅读智能建议
   ├─ 定价建议
   ├─ 预期效果
   └─ 实施步骤

4. 执行定价调整
   ├─ 点击"降价建议"
   ├─ 输入新价格
   └─ 保存更改

5. 监控效果
   └─ 持续跟踪销量变化
```

```
├─ 市场价格走势
├─ 竞争对手采购情况
└─ 供应商砍价空间
```

#### 砍价建议算法

##### 影响因素
```
1️⃣ 采购规模
   ├─ 采购量越大 → 砍价空间越大
   └─ 长期合作 → 砍价率越高

   ├─ 新供应商 → 砍价率高
   └─ 老供应商 → 砍价率低
```

## 技术架构

### 数据结构

#### 进货项 (ProcurementItem)
```typescript
interface ProcurementItem {
  id: string;                           // 唯一ID
  productName: string;                  // 产品名称
  quantity: number;                     // 采购数量
  unitPrice: number;                    // 单位价格
  supplier: string;                     // 供应商名称
  status: 'pending'|'ordered'|'arrived'|'completed';
  date: string;                         // 进货日期
  totalCost: number;                    // 总成本 (quantity × unitPrice)
}
```

#### 比价结果 (ComparisonResult)
```typescript
interface ComparisonResult {
  productId: string;                    // 产品ID
  productName: string;                  // 产品名称
  currentPrice: number;                 // 我们的价格
  marketPrice: number;                  // 市场均价
  competitorPrice: number;              // 竞争对手价格
  priceGap: number;                     // 价格差异
  recommendation: string;               // 定价建议
  priority: 'high' | 'medium' | 'low';  // 优先级
}
```

#### 砍价结果 (NegotiationResult)
```typescript
interface NegotiationResult {
  supplierId: string;                   // 供应商ID
  supplierName: string;                 // 供应商名称
  originalPrice: number;                // 原始价格
  negotiatedPrice: number;              // 砍价后价格
  savings: number;                      // 节省金额 (per unit)
  savingsRate: number;                  // 砍价率 (%)
}
```
